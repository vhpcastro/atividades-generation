PERGUNTAS SOBRE A AULA DE REDE

1. O que são métodos HTTP?
R: São métodos de requisição que indicam a execução uma ação específica.

2. Quais são os métodos HTTP mais utilizados atualmente?
R: GET, POST, PUT e DELETE.

3. Quais são os 2 principais métodos HTTP? Qual a diferença entre eles?
R: GET e POST. GET para requerir consulta enquanto POST para criação de registro.

4. O que são requisições HTTP? Quais os tipos?
R: São requerimentos feitos pelo cliente para iniciar a execução de alguma ação no servidor. GET, POST, HEAD, OPTIONS, PUT, DELETE e TRACE são os tipos de requisições HTTP.

5. Como funciona uma requisição?
R: O funcionamento consiste em enviar uma mensagem para o servidor e nessa mensagem está presente algum método HTTP que indica uma ação a ser executada, assim, o HTTP recebe a informação e retorna uma resposta para o cliente.

6. O que é método GET?
R: GET consiste na requisição de uma consulta para enviar dados ao servidor, consequentemente retornando uma resposta posteriormente.

7. Qual é o protocolo de internet para transferência segura, com uso de certificado digital, utilizado em sites de compras eletrônicas?
R: HTTPS.

8. Quais os principais métodos de uma requisição REST?
R: GET, PUT, POST, PATCH e DELETE.

9. Qual a diferença entre IPv4 e IPv6?
R: IPv4 possui endereços padrões em 32 bits, enquanto IPv6 possui endereços padrões em 128 bits.

10. Faça um resumo sobre os seguintes protocolos: HTTP, TCP, IP, UDP, SMTP e FTP.
R: HTTP funciona no formato Request-Response, ou seja, ele pega informações requeridas pelo cliente e retorna uma resposta para o mesmo.

TCP funciona como um sistema de transmissão de arquivos, enviando dados para os endereços IP.

IP é o endereço de uma aplicação WEB, basicamente sua identificação dentro da internet.

UDP é semelhante ao TCP, permitindo o envio de dados em um pacote, porém sem a garantia de que as informações chegarão ao destino, já que é mais simples e não possui conexão.

SMTP funciona na emissão de e-mails e garante que eles chegem ao destinatário através da configuração das regras de comunicação entre servidores.

Por fim, o FTP permite o envio e recebimento de arquivos dentro de uma rede de computadores independentemente de hardware, desde uma local à internet.
